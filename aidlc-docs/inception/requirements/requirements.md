# 필그림시니어랜드 기업 소개 홈페이지 - 요구사항 문서

## 인텐트 분석 요약

- **사용자 요청**: 필그림시니어랜드 운영을 위해 설립된 특수목적법인의 기업 소개 홈페이지 개발
- **요청 유형**: New Project (Greenfield)
- **범위 추정**: 다중 페이지 웹사이트 (멀티 섹션 + 문의 폼)
- **복잡도 추정**: Moderate (정적 콘텐츠 위주 + 문의 폼 백엔드 처리)

---

## 프로젝트 개요

**회사명**: 필그림시니어랜드 (특수목적법인)
**사업 내용**: 대한민국의 건강하고 행복한 노년을 준비하는 노인복지주택사업 추진
**목적**: 기업 소개 및 입주 관심 고객 유입을 위한 공식 홈페이지

---

## 기능 요구사항 (Functional Requirements)

### FR-01: 페이지 구성 (전체 섹션 포함)

| 섹션 | 설명 |
|------|------|
| 홈 (Hero) | 브랜드 첫인상, 핵심 메시지, CTA 버튼 |
| 회사 소개 | 설립 배경, 비전, 미션, 특수목적법인 소개 |
| 사업 소개 | 노인복지주택사업 내용, 추진 방향 |
| 시설/단지 소개 | 필그림시니어랜드 시설 정보, 이미지 갤러리 |
| 입주 안내 | 입주 조건, 절차, 비용 안내 |
| 뉴스/공지사항 | 회사 소식, 공지 목록 (정적 콘텐츠) |
| 문의하기 | 상담 신청 폼 + 연락처 안내 |
| 개인정보처리방침 | 법적 필수 페이지 |

### FR-02: 문의/상담 폼
- 입력 필드: 이름, 연락처(전화번호), 문의 내용
- 서버 사이드 입력값 검증 (JSP/Servlet)
- 폼 제출 시 이메일 발송 또는 DB 저장
- reCAPTCHA v3 적용 (스팸/봇 방지)
- 제출 완료 후 감사 메시지 표시

### FR-03: 내비게이션
- 상단 고정 네비게이션 바 (로고 + 메뉴)
- 모바일 햄버거 메뉴
- 스크롤 시 섹션 하이라이트 (Active 상태)
- 부드러운 스크롤 이동 (Single Page 또는 Multi Page)

### FR-04: 뉴스/공지사항
- 정적 HTML 파일로 관리 (CMS 없음)
- 목록 페이지 + 상세 페이지 구조

---

## 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 반응형 디자인
- 지원 해상도: 모바일(320px~), 태블릿(768px~), 데스크탑(1280px~)
- 모든 섹션 반응형 레이아웃 적용
- 터치 친화적 UI (시니어 사용자 고려 - 충분한 버튼 크기, 가독성 높은 폰트)

### NFR-02: 디자인 방향성
- 세련되고 모던한 스타일 (요즘 트렌드)
- 심플하고 깔끔한 레이아웃
- 시니어 복지 업종에 맞는 신뢰감 있는 톤앤매너
- 브랜드 자산: 로고 보유 (컬러 가이드 없음 → 로고 기반으로 컬러 팔레트 도출)
- 큰 타이포그래피, 충분한 여백, 고품질 이미지 활용

### NFR-03: 성능
- 페이지 초기 로딩 3초 이내 (LCP 기준)
- 이미지 최적화 (WebP 포맷, lazy loading)
- CSS/JS 최소화 (minification)

### NFR-04: 접근성
- 시니어 사용자 고려: 최소 폰트 크기 16px 이상
- 충분한 색상 대비 (WCAG AA 기준 참고)
- 키보드 네비게이션 지원
- 이미지 alt 텍스트 제공

### NFR-05: 언어
- 한국어 단일 언어

### NFR-06: 런칭 목표
- 1~2개월 내 런칭 목표

## 기술 스택 (Technical Stack)

### 호스팅 환경
- **호스팅**: Cafe24 JSP 호스팅
- **WAS**: Apache Tomcat 10.0.x
- **JDK**: JDK 11
- **JSP/Servlet**: JSP 3.0, Servlet 5.0
- **DB**: MariaDB 10.x (UTF-8)

### 프론트엔드
- HTML5 + CSS3 + Vanilla JavaScript (또는 경량 라이브러리)
- 반응형: CSS Flexbox / Grid
- 빌드 도구: 없음 또는 최소화 (정적 파일 직접 배포)

### 백엔드
- JSP + Servlet (문의 폼 처리)
- JavaMail API (이메일 발송) 또는 MariaDB 저장
- reCAPTCHA v3 서버 사이드 검증

---

## 보안 요구사항 (Security Requirements)

### SEC-01: 전송 보안
- HTTPS 적용 (SSL 인증서)

### SEC-02: 입력값 보안
- 서버 사이드 입력값 검증 (XSS 방지)
- SQL Injection 방지 (PreparedStatement 사용)
- CSRF 토큰 적용 (문의 폼)

### SEC-03: 스팸 방지
- Google reCAPTCHA v3 적용

### SEC-04: 보안 헤더
- X-Content-Type-Options, X-Frame-Options, Content-Security-Policy 설정

### SEC-05: 개인정보
- 개인정보처리방침 페이지 필수 포함
- 문의 폼 수집 항목 최소화 (이름, 연락처, 내용만)

---

## 테스트 요구사항

- 단위 테스트: Servlet/JSP 로직 (JUnit 5)
- E2E 테스트: 주요 사용자 흐름 (문의 폼 제출 등)
- 크로스 브라우저 테스트: Chrome, Safari, Edge
- 반응형 테스트: 모바일/태블릿/데스크탑
- Property-Based Testing: 생략 (단순 홈페이지에 불필요)

---

## 제약 사항 (Constraints)

- CMS 없음 - 콘텐츠 변경 시 파일 직접 수정
- Cafe24 JSP 호스팅 환경 제약 준수
- 로고 외 브랜드 자산 없음 (컬러/폰트 새로 정의 필요)
- 호스팅 도메인 연결 필요 (도메인 보유 중)

---

## 성공 기준 (Success Criteria)

- 모든 섹션 정상 렌더링 (모바일/태블릿/데스크탑)
- 문의 폼 정상 작동 (입력 검증 + 제출 처리)
- 페이지 로딩 3초 이내
- HTTPS 적용 완료
- 개인정보처리방침 페이지 포함
