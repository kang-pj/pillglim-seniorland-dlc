# 비즈니스 규칙 (Business Rules)

## 공지사항 (Notice)

| 규칙 ID | 규칙 | 처리 |
|---------|------|------|
| NR-01 | 목록은 최신순 정렬, 페이지당 10건 | 서버 사이드 페이징 |
| NR-02 | is_published = 1 인 항목만 노출 | WHERE 조건 |
| NR-03 | 홈 페이지에는 최신 3건만 미리보기 | LIMIT 3 |
| NR-04 | 관리자만 등록/수정/삭제 가능 | 세션 권한 체크 |

## Q&A

| 규칙 ID | 규칙 | 처리 |
|---------|------|------|
| QR-01 | 익명 고객이 질문 등록 가능 (닉네임 선택 입력) | 미입력 시 "익명" 기본값 |
| QR-02 | 비밀글 설정 시 비밀번호 입력 필수 | 클라이언트 + 서버 검증 |
| QR-03 | 비밀글은 목록에서 제목 "비밀글입니다" 표시 | is_secret 플래그 |
| QR-04 | 비밀글 상세 조회 시 비밀번호 확인 페이지 표시 | 비밀번호 일치 확인 |
| QR-05 | 관리자는 모든 비밀글 조회 가능 | 세션 권한 체크 |
| QR-06 | 답변 완료 시 is_answered = 1, answered_at 기록 | 관리자 답변 등록 시 |
| QR-07 | 목록은 최신순 정렬, 페이지당 10건 | 서버 사이드 페이징 |
| QR-08 | 질문 내용 최대 1000자 | 서버 사이드 검증 |

## 온라인 문의 (Contact)

| 규칙 ID | 규칙 | 처리 |
|---------|------|------|
| CR-01 | 이름: 필수, 최대 50자 | 서버 검증 |
| CR-02 | 연락처: 필수, 한국 전화번호 형식 (010-XXXX-XXXX 등) | 정규식 검증 |
| CR-03 | 문의 내용: 필수, 최대 1000자 | 서버 검증 |
| CR-04 | reCAPTCHA v3 score >= 0.5 통과 필수 | 서버 사이드 검증 |
| CR-05 | XSS 방지: 모든 입력값 HTML 이스케이프 | ValidationUtil |
| CR-06 | CSRF 토큰 검증 | 폼 제출 시 |
| CR-07 | 제출 성공 시 담당자 이메일 알림 발송 | EmailUtil (비차단) |
| CR-08 | 이메일 수신자 주소는 application.properties에서 관리 | 설정 파일 |
| CR-09 | IP 주소 기록 (스팸 방지 참고용) | 서버에서 자동 수집 |

## 관리자 인증 (Admin Auth)

| 규칙 ID | 규칙 | 처리 |
|---------|------|------|
| AR-01 | 관리자 로그인: username + password | BCrypt 비교 |
| AR-02 | 로그인 성공 시 세션에 admin 정보 저장 | HttpSession |
| AR-03 | 관리자 페이지 접근 시 세션 유효성 확인 | Filter |
| AR-04 | 세션 타임아웃: 30분 | web.xml 설정 |
| AR-05 | 로그아웃 시 세션 무효화 | session.invalidate() |
