## Question 1: 공지사항 / Q&A 데이터 입력 방식

공지사항과 Q&A는 DB에서 조회하기로 했는데, 초기 데이터는 어떻게 입력할 예정인가요?

A) DB에 직접 SQL로 입력 (개발자가 관리)
B) 나중에 간단한 관리자 입력 페이지 추가 예정 (지금은 SQL로)
C) Other (직접 설명해 주세요)

[Answer]: C Q&A , 공지사항은 모두 게시판 형식이라 공지사항은 관리자만, Q&A는 익명 고객이 남길 수 있도록

## Question 2: 온라인 문의 알림 이메일 수신자

문의 폼 제출 시 알림 이메일을 받을 담당자 이메일 주소 설정 방식을 선택해 주세요.

A) 코드/설정 파일에 고정값으로 지정 (배포 후 변경 시 재배포 필요)
B) DB 또는 properties 파일에서 관리 (배포 없이 변경 가능)
C) Other (직접 설명해 주세요)

[Answer]: B properties에서 진행

## Question 3: Q&A 운영 방식

Q&A 페이지를 어떻게 운영할 예정인가요?

A) 자주 묻는 질문(FAQ) 형태 - 질문/답변 쌍을 미리 DB에 등록해두고 표시
B) 고객이 질문을 올리면 관리자가 답변하는 게시판 형태 (나중에 관리자 기능 추가)
C) 지금은 FAQ 형태로 시작, 나중에 게시판으로 확장
D) Other (직접 설명해 주세요)

[Answer]: B

## Question 4: 평면소개 페이지 콘텐츠 형태

주거공간 소개 > 평면소개 페이지는 어떤 형태로 구성할까요?

A) 이미지 갤러리 형태 (평면도 이미지들을 슬라이더/그리드로 표시)
B) 타입별 탭 구성 (A타입, B타입 등 탭으로 구분하여 각 평면도 표시)
